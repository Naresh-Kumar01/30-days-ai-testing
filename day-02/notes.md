\# Day 02 - Prompt Engineering for Software Testers



\## Objective



Learn how to write effective prompts that generate accurate and high-quality outputs for Software Testing tasks using AI.



\---



\# What is Prompt Engineering?



Prompt Engineering is the practice of writing clear, structured, and detailed instructions for AI models to produce better responses.



\*\*Simple Formula:\*\*



```

Better Prompt = Better Output

```



\---



\# Why Prompt Engineering is Important



A well-written prompt helps AI:



\- Understand the context

\- Produce accurate responses

\- Reduce irrelevant information

\- Save time

\- Improve productivity



For Software Testers, Prompt Engineering can be used to generate:



\- Test Cases

\- Playwright Automation Scripts

\- API Test Scenarios

\- SQL Queries

\- Bug Analysis

\- Root Cause Analysis

\- Test Data

\- Code Reviews



\---



\# Prompt Structure



A professional prompt should include:



1\. Role

2\. Objective

3\. Context

4\. Requirements

5\. Constraints

6\. Output Format



Example:



Role

→ Senior Quality Analyst



Task

→ Generate Login Test Cases



Context

→ E-commerce Application



Requirements

→ Login using Email \& Password



Constraints

→ Cover Positive, Negative, Boundary, Edge Cases



Output

→ Markdown Table



\---



\# Basic Prompt



```

Write Login Test Cases.

```



Problems:



\- No role

\- No context

\- No requirements

\- Generic output



\---



\# Improved Prompt



```

Act as a Senior Quality Analyst with 10+ years of experience.



Generate comprehensive test cases for the Sign In functionality of an e-commerce application.



Cover:

\- Positive

\- Negative

\- Boundary

\- Edge Cases

\- Security

\- UI

\- Usability



Output:

Markdown table including Test Case ID, Scenario, Steps, Expected Result, Priority and Test Type.

```



\---



\# Key Learnings



\- AI performs better with detailed prompts.

\- Always define the AI's role.

\- Provide sufficient business context.

\- Mention constraints and expected coverage.

\- Specify the desired output format.
